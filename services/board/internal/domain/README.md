# Domain

**Цель:** хранение сущностей, value objects и агрегатов.

## Правила

- Бизнес-логика только здесь.
- Usecases не реализуют правила, только orchestration.
- Пример: `Board.MoveCard()` проверяет все инварианты.
- Никаких внешних зависимостей (БД, Redis, очереди).
- Aggregate Root управляет Value Objects (Column, Card).
- Ошибки и типы только domain (`ErrAccessDenied`, `ErrCardNotFound`).

## Не делаем здесь

- Доступ к БД, Redis или очередям.
- Вызовы API других сервисов.
- Оркестрацию сценариев (usecase).
- Любую бизнес-логику, относящуюся к другим агрегатам.
