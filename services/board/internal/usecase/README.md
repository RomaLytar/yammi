# Usecase

**Цель:** сценарии, оркестрация domain + репозитории.

## Правила

- Оркеструет domain, не содержит бизнес-правила.
- Пример: вызывает `board.MoveCard()`, публикует событие.
- Взаимодействие только через интерфейсы репозиториев.
- События публикуются только после успешного выполнения сценария.
- Authorization проверяется здесь перед вызовом domain.

## Не делаем здесь

- Прямой доступ к БД, Redis или внешним сервисам.
- Реализацию бизнес-логики domain.
- Изменение Value Objects напрямую без Aggregate Root.
